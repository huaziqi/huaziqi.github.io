  3. 排序算法的时间复杂度（考的比较少）

排序算法的稳定性

稳定性是指排序前两个元素a1 = a2，a1在前。排序过后，倘若a1始终在前，则算法是稳定的，否则是不稳定的。

稳定的

冒泡排序、插入排序、归并排序、基数排序（简写）

不稳定的

堆排序（简写）、快速排序、希尔排序、选择排序

简评《黑客与画家》

《黑客与画家》简评

本书的作者保罗•格雷厄姆，被誉为硅谷创业之父。本书中，他提出了许多观点，初看与全书主题无关，当回头再看时竟全都服务于这个标题，是一本优秀的教育书籍。文章以参叙参议的形式，阐述了“黑客”这个名词的较为“公正”的含义，作者在其中也传授了一些自己的创业经验。

作者善用类比来更为详尽地表达自己的看法。书名便是如此——黑客与画家，将黑客与画家进行类比，指出黑客与画家的高度相似性：他们都在创造美的事物，他们所做的事也许都会在许多年后才被流传、称赞，他们都是一个时代拥有明锐嗅觉的先驱。这样的类比还有例如：书呆子与“怪人”，编程与挑选衣服等。

其次，作者善用亲身经历来论证观点。例如在第一章中，作者阐述“书呆子”的各种特点时，以自己的亲身经历为基础，使得叙述更加可信，也更易与读者产生共鸣。包括后面在描述编程语言的优劣时，列举了当初自己的创业经历。

接着开头所说到的分论点服务于书名，全书共15大章，每一大章又分许多小标题。看似毫无关联，实则作者讲述了一个优秀“黑客”所具有的潜质。将每一个标题用一条线串起来，浑然成了“一本”书。

网络组成与构建

网络的组成与构建

由于选到了一个比较难的讲题（必修二），做一个基本知识的整理

通常我们上网都会输入一个网址，例如：

http://www.gov.cn/index.htm

http为协议名，www.gov.cn是网站服务器的地址，index.htm则是网站服务器里的某个文件。这其中我们需要通过电脑来访问网站，我们的电脑就是终端。访问网站的软件，例如Chrome，Firefox等浏览器，这些浏览器就是网络软件。

来看下面这张图片。

当终端与服务器不在同一个局域网内时，我们如何获取服务器内容呢？我们可以将一个局域网看做一个小区，网关是门卫，那么数据信息系统就充当了快递员的作用，把其他小区的东西运过来。

网络的组成

从狭义的角度来说，网络似乎是个虚无缥缈的东西。但如果我们从广义的角度来看，组成网络的实体其实在我们身边无处不在。

计算机系统

终端

想要上网，我们自己肯定得先有一台“设备”，以便于我们与其他“设备”形成联系。这里的“设备”可以是电脑，手机等等，专业一点的叫法就是“终端”。

终端是网络中数量大、分布广的设备，是用户进行网络操作、实现人机对话的工具。

服务器

服务器负责数据处理和网络控制，并构成网络的主要资源。

我们之前用flask搭建网页，并且把自己的电脑变成了一个服务器来使。服务器相对于我们的电脑，稳定性、安全性、性能等方面要求更高，它可以分为大型机，中小型机等。假如我们用机房电脑发布一个网站，万一极域把网给掐了，或者北师大又停电了，就💥了。通过服务器我们可以获取网络上的各种资源，但服务器再强大也会有崩溃的一天。经典服务器宕机的事例就有前不久的b站崩溃。

数据信息系统

主要由传输介质和网络互联设备等组成。

传输介质

顾名思义，指的是各种传输数据物理通道，也就是各种线（有线与无线）。

  • 双绞线

  • 同轴电缆

  • 光缆

无线的有无线电微波信号和红外信号等

网络互联设备

调制解调器、路由器、交换机。

调制解调器（Modem），也就是我们平时所说的“猫”，是 modulator（调制器）和 demodulator（解调器）的合成，就是“调制解调器”。他将传输进来的模拟信号转化成数字信号，这是解调的功能；同时他还能将我们要发出的数字信号转换成模拟信号便于传输，这便是调制的功能。

路由器（Router）又称网关设备（Gateway），它实现了多设备一起上网的功能，具备无线WiFi网络供移动设备一起上网。倘若没有路由器，家里拉进来的光纤或网线只能供一台设备使用。

交换机用于拓展路由器的接口。

==以上说的都是硬件层面的，接下来则是软件==

网络软件和网络协议

网络软件

网络软件：授权给用户，使用户能访问网络资源；管理和调度网络资源，提供网络通信和服务。网络软件包括网络操作系统、通信软件以及管理服务软件等。

网络操作系统负责网络软硬件的管理以及网络通信和任务的调度，提供用户与网络之间的接口，常见的UNIX、Windows Server、Linux。

网络协议

就像是一门公认的语言，网络协议为不同的终端、不同的网络之间的互相识别与通信提供了准则，是计算机网络正常工作的基础。

国际标准化组织（ISO）在1978年提出了“开放系统互联参考模型”，即著名的OSI/RM模型。协议就是不同设备这些层之间传输数据的标准

OSI/RM模型
应用层：浏览器，邮件等。有HTTP协议。
表示层：将数据进行编译，加密，压缩等等。
会话层：负责身份验证、授权，数据包的追踪。
传输层：TCP协议，管理被传送内容的完整性。
网络层：IP协议，负责将信息从一个地方传送到另一个地方。
数据链路层：AP协议，将传输的信息转换成人类能识别的内容。
物理层

网络的构建

通过数据通信系统将计算机系统连起来，并在网络软件和协议管理下工作。构成的最小网络是局域网，无数个局域网通过数据通信系统汇聚到运营商，运营商之间再互联，就形成了互联网。

确定网络结构

我们通常使用树形的网络结构，这样的好处在于易于隔离故障。

连接网络设备

WAN口与LAN口的区别就在于WAN口连接外网LAN口连接内网

  • 将ISP光纤接入调制解调器
  • 将调制解调器（猫）连接到路由器的WAN口
  • 有线终端用双绞线连接到路由器LAN口，无线终端就连WLAN

接入互联网

通过路由器将局域网的私有地址转化为公有地址。

  • （1）设置路由器连接因特网的方式，一般用PPPoE拨号上网，在WAN口设置中输入用户名密码即可（还有静态地址接入、动态地址接入）
  • （2）启用网络路由器的网络地址转换功能，默认开启

终端设备接入路由器需要一个和路由器同一网段的IP地址。路由器、交换机、服务器等设备可以使用静态地址PC机，网络电视机顶盒，数字摄像头等可以用动态地址，需启用DHCP动态主机配置协议。

移动设备要接入，必须开启路由器的无线网络功能。需要设置SSID（网络名字），加密方式、共享密钥。移动终端打开WLAN，搜索到无线网络，输入账号密码即可连接。

网络连通性测试

架设完成后对网络进行连通性测试，确定网络是否正常工作。可以通过终端浏览网页是否正常等简单方法来测试

富山感遇2023

富山感遇

这两天用PySimpleGUI做了个打卡升级的自动化办公软件。没想到因为没日没夜在改bug，导致本来已经伤愈的右手又肿起来了。今天早上要去富山，我跟我妈说了这件事，本来说要带去的电脑被扣留在了家里。

富山是哪

从我记事起，几乎每年都会去的地方。外婆的老房子，石板下流过的清澈溪水，武侠剧里的竹林，再往上爬，是寺庙香火缭绕的地方。总之，就是这样一个世外桃源，每一次来都会有不一样的感受与遇见。

上山

上山前跟我妹打了一架，没错，是打了一架。具体的情况我不愿多说，原因有多方面，最重要的一方面应该就是情绪。对于陌生人，我们应以平等的姿态面对，别人打你一巴掌，你也要打回去一巴掌，甚至两巴掌。否则岂不是别人欺侮了。对于重要的人不应如此。克制自己的情绪。

人类不喜欢用暴力解决问题，因为暴力是最有效也最有害的方法，它损害钱财，损害感情，甚至生命。

上山的旅程很是艰辛，我妹吐了好几次，我也有些不适。直到我发现晕车的本质所在。

初中时期我们就学过：大脑中有个器官能判断自己所处的位置。晕车正是因为大脑无法正确判断接下来的走向。因此只要一直看着前面，假装自己是司机，大脑就会清楚地知道接下来的位置，就不会晕车了。亲测有效。这大概就是司机不晕车的原因了。

第一天

上午晚上没有什么感受，因为一直在玩手机。

远方的竹林不再拨动我的心弦，我害怕他是不是断了。

下午两三点钟时去拜访了一下就未谋面的娘舅。路上我睡着了，大抵是昨晚打cf累了。到了之后，娘舅早早便出来迎接。大大小小的饮料、土特产从后备箱拿出。

屋子黑漆漆的，屋里基本没什么家具。来到卧室，一群人围着火盆坐下。不一会便端来了一碗碗荔枝汤，还挺甜。

电视机前，我妹还是一如既往跟我抢遥控板。

喝汤的时候，因为荔枝有核嘛，我妹就懂事地把垃圾桶拿过来放在中间。于是众人纷纷夸奖她有眼力见，而这眼力见又是我总被父母数落的东西。一个4、 5岁的小孩，竟像个大人一般，吃饭记得关电视，吃饭的时候夹菜给别人，想我妈一样数落我。而我很难做到这些。也许是生性如此，我不愿展现自己，却又希望得到表扬，于是外人常说我很老实，于是我也慢慢接受这个标签。但是只有我自己知道：我没有他们所说的那么“老实”。

我也会有小心思，甚至有的时候总会想得太多。“妹妹把我的话给抢完了”，我妈是这样说的。然而在学校，我总是尽可能展现自己，而非像是在长辈亲戚面前那么矜持。

在我的心里，亲戚不过是系统默认好友，而实际上，你跟他们中的大部分根本不熟。我是一个讨厌封建礼教的人，也就是孩子不能顶嘴，遇到长辈要叫什么的。求神拜佛之类也只有在我小时候懵懂无知时才相信。至少现在我是不相信佛祖的存在，至少神并非生物。

不一会，舅公拿着两个红包笑嘻嘻地走了过来。他说：“你现在还没赚钱，这压岁钱先收着。等你以后挣钱了，记得回来看看我就好。”推搡一番后，我便收下了。他所说的那番话确实令我有所感触。明明我印象中只与他有过两面之缘，他却拿出800元的红包。这或许就是老人家心中的铁律。一年攒下几个钱，过年就全发出去了。

如果几年之后我仍记得他，我一定会回去看看。

第二天

一大早起来，外公说要带我们去山里看看。拿着外公自制的登山棍（两根细竹棍），我走在前面。此时，外公口中人一样大的狼狗吓了我们一跳，连忙把棍子收起来，换了条路走。

路过一排排民居，远方的竹林绵延成山。

外公肩扛锄头，腰间木质“腰带”别着镰刀，头上还带着毡帽。平日里他不怎么说话，跟外婆比起来存在感自然较低。先前住在路桥的时候，我还听说过外公的光辉事迹：骑着自行车跟鬼火少年飙车，属实是人老心不老了。有人问问题时，他总能尽可能答上，没人回应也会和蔼地笑笑，算是缓解尴尬。

走过竹林夹道的山间小路，突然间，经过一个上坡后，眼前的景色瞬间开阔。一片片的田地错落有致，让人忍不住想躺在田里一整天。这简直和我想象中的晚年生活别无二致。住在偏僻的山里，乡镇不大，人们都互相认识。拥有自己的田地两亩，时不时扛着锄头来看看，翻翻土。偶尔去赶赶集，还可以把自己种起来的白菜萝卜摆到集市上，不为赚钱，但求清闲。

继续往上走是杨梅林。杨梅林中竟有一间上了锁的小屋，现在大抵没人在的。

一路走一路聊，正尽兴之时，一通电话扫了兴。不知道是谁打来的，只看见我妈脸上不怎么高兴。似乎是来了些不速之客，我们只好下山。下山路的一个转角，竟猛地有只类似于鸡的野禽飞了出来，羽毛还挺好看，算是长见识了。

中午

大概知道了是一群上山拜佛的人过来蹭饭，大约有8、 9个。知道这个消息倒还没什么，直到中午一群人莫名其妙就来了。其中有个小孩大约小学一年级的样子。

我和我妹还在外面玩土，屋里叫饭好了。本想着可以吃饭了，没想到那一群人整整齐齐算是把圆桌围满了，真是迅雷不及掩耳之势，都不知道他们什么时候进来的。想给我妹乘碗饭，没成想又有两个人挤过来，真的是“无地自容”了。见此情形，爸妈就说让我们先别吃，等他们吃完。我心里自然是不高兴，但是没办法，我也不想跟群不速之客抢食吃。

吃着他们吃剩下的饭菜，心里总不是滋味。没想到，这群人的头头，拿着500块钱交给外婆就说要走了。什么纯纯的逆天，当这里是饭店呢？的亏还是个老板，这么一桌子菜。更何况他妈跟外婆还是姐妹。属实难绷。当我们服务员怎么着，傻逼。

下午写了会作业。我妹说还想住一晚就多住了一晚。

第三天

一大清早起来吃完早饭。收拾收拾我们就走了。

第一件事不是回家而是去医院。前两天手不是肿了吗，去了黄岩的一家医院看。没拍CT前就觉得那个医生不对劲。我手刚断的时候，那家椒江的医院拍的都是普通片，而非CT。结果这个医生忽悠着什么CT更清楚，但是最后片拍出来看起来的效果跟普通片没什么区别。

看了片子，他竟然又说我骨头错位了，如果不保护好可能会导致手的什么功能障碍。这不仅把我吓了一跳，我妈也开始骂我了。而我的手明明已经丝毫没有痛觉了。

然后他给我开了点药，医保保完60。我就寻思那种止痛贴对于骨头错位有用吗。

这次的误判可以说是之后几天家庭关系走向冰点的导火索。我妈一口咬定是我玩电脑玩多了，于是缩短了我使用电脑的时间。明知冤枉，但我妈的性格我还是清楚的，她咬定的东西绝不会退让。另一方面，我对于自己的判断也产生了怀疑，并开始担心未来这手对于我的影响。

之后发生了很多事，本都可以和平解决，但由于我的心情极差，差到开始思考人生的意义。以至于家庭矛盾尖锐（当然这也与家庭教育方式有很大的关系）。

直到之后去了一家骨科医院，才为我洗清冤屈。

对于此事，我不知道那个医生是故意想捞钱还是本身技术不高，我只知道他对他的工作没有负责任的态度，更没有抱着一颗热爱的心去工作。

B. Matrix of Differences

题意

对于一个n * n的矩阵，计算每一个相邻的数字之差的绝对值，输出绝对值覆盖范围最广的方案。

例如当 n = 2

::: tips

|1 - 4| = 3、 |1 - 3| = 2、 |3 - 2| = 1、 |4 - 2| = 2。1 2 3 都有，是最佳方案。

:::

分析

猜一下结论，最好的方案绝对值必然从[1, - 1]都有。

所以1和必须挨着。不然 - 1无法产生。

接下来就毫无思路了😔

看题解

找规律题。

有一种办法感觉跟我稍微想到了一点儿。就是一个小一个大，例如n = 3。第一行从左往右1 9 2，第二行从右向左8 3 7，第三行从左向右 4 6 5，就是一个蛇形矩阵。

原理是：最大程度的充分接触.|1 - | = - 1， | - 2| = - 2， | 2 - ( -1) | = - 3 ……以此类推，这条贪吃蛇共有8条线，产生了要求的8个数字。

规律有了，代码就好写了。

什么是左儿子右兄弟

首先，对于这样一棵多叉树，每个节点连接的有子节点，兄弟节点，父节点。

例如，1号节点有2 3 4三个子节点，这三个节点在构造二叉树时就被放在左边，这就是左儿子。

2号节点有3 4两个兄弟节点，所以在构造时，这两个节点都放在了右边，这就是右兄弟。

因为 第一个 与 父节点 连接的兄弟节点有三个，所以说有三种以上可能。这样子研究起来肯定很复杂。我们换个角度，把这些二叉树看成是一个一个链表。

链式前向星

来个大点的图，将其改为二叉树，其中一种方案是这样的

运用链式前向星的思想，每个节点创建一个单链表，链表存储的是父节点与子节点。1 是根节点，2 3 4 5是子节点，被放在该链表内。可以看到我用虚线隔开的三个区域，中间的区域就包含着2 3 4 5，6 7是2的子节点，8是4的子节点。从上图应该能够看出一点规律来：兄弟节点被排成了一列。

这可以说是一个有趣的性质，启发我们使用链式前向星存图。

递归与贪心

如何求得最大长度？获取每个节点子树高度。上图中1的子节点中谁的子树高度最高？是2。

我们如何求子树的最大高度？遍历

遍历1的子节点，对于1的每个子节点，要求它们的子树高度，就要遍历它们的子节点，直到到底，才一级一级返回他的子树高度。这就类似于一个递归的过程。

那兄弟节点之间的顺序呢？开头说到多个兄弟节点会有多种排法，既然我们都求出了拥有最大的子树高度的节点，那在给他加点高度，也就是放到最下面，那岂不是锦上添花。所以不用考虑过多，只要把子树高度最高的加上兄弟节点数就好

Hello hexo！—— hexo + github部署教程

前前后后搭了有两三天，终于算是搭好了。

写一写搭建过程，搭建方法，遇到的问题……反正这篇博客成分复杂

  • 对历程不感兴趣的可以直接跳到搭建方法

搭建历程

开始有搭建的想法大约是高一下学期，那个时候接触了一些技术博客，并尝试着自己写博客。在感受到写博客的快乐与美好之后，搭建一个个人博客网站的想法被一个视频引起。

第一个想法

是一个知乎的视频，它具体教授了如何使用hexo与github pages 搭建博客。可惜我当时对于github完全没有接触，更不知道如何去使用。那个视频中还使用vim来编辑配置文件，当时我看的是一脸懵逼：他是如何保存那个文件的？ctrl s？不是，于是我在视频底下问他，却迟迟没有得到回复。第一次博客搭建计划失败

money

在之后，我接触了其他的搭建博客的方法，例如WordPress。然而使用WordPress需要自己去买域名（一年30~50）和服务器（一年100往上）。当时还在学校，资金没有来源，何况这也不是一笔小数目。那个时候在csdn还是博客园上写博客（先是在CSDN，但是由于它不可美化且页面广告太多，也不美观，就转去了博客园）也就安于现状了。

这期间学会了如何使用github，Markdown等工具，美化了自己的博客园，算是为之后做铺垫。

vim

高二下学期，经过互联网上各路网友的推荐，我去学习了vim，对于它的各种快捷键略知一二了。然而使用起来还是不习惯，平时就没有过多使用。但是我终于知道了vim编辑器的保存方式是 :wq ，于是兴冲冲地再去尝试，却被卡在了最后一步hexo d， 怎么都无法上传，网上找了各种方法（其实就两种）也没用。再一次失败了……

转机

昨天，我再一次尝试搭建博客。这一次，我不能输！

既然github不行，那就用gitee。于是找了网上hexo + gitee的教程。不同于之前github的教程的是，之前用的是repo，而这次使用的是ssh。碰巧看到一个视频就是讲之前的hexo博客搭建实际上已经过时了。

考虑到gitee是国内网站，访问速度快，github访问不太稳定，于是想要使用gitee pages，可没想到，gitee居然还要实名认证：），实名时间还要1到2个工作日，我寻思有那么多人实名吗，我在博客园申请json权限，5分钟就通过了。更何况实名似乎根本就没有意义，国内的网站、软件环境都是这样，你就这么怕我们说什么东西吗？？

还有为什么国内每个稍微大一点的网站都要手机号验证？真的蚌埠住，就这么不喜欢邮箱？邮箱不是都绑定过手机号了吗？

于是转用了github，访问的问题是使用了netlify免费代理。

搭建方法

以下命令均在cmd(windows)/控制台(mac)使用

本地hexo环境搭建

  • 首先下载git与node（文字上有超链接）

使用git --version检查git是否安装

使用node -v检查node是否安装

  • 之后使用node安装hexo：

输入 npm install hexo-cli - g 安装hexo

开始搭建hexo

hexo init 文件夹名称（随你写）

比如hexo init huaziqi，就会创建一个叫”huaziqi“的文件夹，该文件夹保存在根目录，找不到可以用计算机搜索功能找一下。

之后输入cd 你的文件名（你刚刚创建的那个）

cd 文件夹的意思就是进入文件夹

cd ..是退出文件夹

ls是输出该文件夹下的所有文件夹和文件

这些都是控制台的基本操作

进入文件夹之后输入npm install，安装基本文件

hexo中的基本操作

hexo g 相当于编译，把你所有对于文件夹中的改变识别到，有错误的话会报错。

完成之后输入hexo s就可以在本地访问你的网站

加载完应该是这样，进入 http://localhost:4000/ 就是你的博客在本地的样子。

==先别急着优化博客==，确认它能够被流畅访问先，不然失败了的话就很难受

部署博客到github

npm install hexo-deployer-git --save 下载上传的插件

获取ssh

据我所知，现在部署博客有两种方法，一种是用repo，另一种使用ssh密钥。之前失败就是使用的前者。所以现在是用后者。

  • 注意，这些操作都是在你创建的hexo文件夹里进行的

这两句进行后不会返回任何东西

如果他问你Enter file in which to save the key (/Users/TMnoL/.ssh/id_rsa): ,你就填id_rsa

接着他会让你输入密码，注意这个密码自己==一定要记住==，之后每一次部署你的博客都需要你输入一遍

然后它会在该文件夹内产生两个文件：id_rsa、id_rsa.pub。

把这两个文件移动到上一级目录中的.ssh文件夹。

这个文件夹是隐藏的，如何查看隐藏文件夹，mac是command+shift+.，这东西自己百度就行。

绑定ssh到github

然后去github（未注册自行注册，用户名不要乱取哦）

之后点击自己的头像（右上角），点击settings。

在左列找到 SSH and GPG keys

点击new SSH key，把刚刚生成的id_rsa.pub里的东西全都粘贴到下面的 key 框里，title随便填就好。

Add SSH key

新建github库

建库不会自己去学。

注意：库的名称一定要是自己github的用户名+.github.io

比如我的用户名是huaziqi，那就要建叫 huaziqi.github.io，这也是之后你的网站名（当然自己有别的域名是可以改的）

仓库类型当然是public，公开的。

建好仓库以后打开仓库点击那个code，在里面吧SSH代码复制下来。

FINAL

回到创建的文件夹，找到_config.yml文件打开。找到deploy，像我一样这么填。repo里填的是刚刚粘贴的SSH代码。保存文件。

到控制台输入hexo g，加载完再输入hexo d，输入生成id_rsa.pub时自己输入的代码，等待片刻就成功了。

访问你的用户名+github.io，其实就是你的仓库名。我的是huaziqi.github.io，欢迎来访问。
